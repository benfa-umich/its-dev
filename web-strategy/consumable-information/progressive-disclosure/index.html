Progressive Disclosure

Progressive disclosure puts key information front and center, allowing users to obtain detailed information only if they want.

The concept of disclosing information progressively is designed to "help maintain the focus of a user's attention by reducing clutter, confusion, and cognitive workload. This improves usability by presenting only the minimum data required for the task at hand." (source: Wikipedia on Progressive Disclosure).

See this talk on Progressive Information Disclosure for more on how to apply it to our work.

Best Methods

  • The Inverted pyramid - Often used in journalism, this method involves placing the most important information first, then successively less important information after.
  • Links to further content on another page are the most common and effective way to progressively disclose. A brief summary that links to a page with more information is the best way to provide the type of response users seek.

Special Use Methods

Toggles, accordions, tabs and other such widgets hide info on a page and require the user to actively click to see it.

Never use these techniques just to make a page look less text-heavy. User studies show that users get annoyed when having to click to view info that most people will want to read anyway.

Use them only when a piece of text is not of interest to most users; everyone can see the linked text and click if they need to see more.

Good example:

  • a set of like items - if most users will only be interested in 1 or 2 of the items, let them choose those items for which they want to see detail

Bad examples:

  • in a set of like items, if most users are likely to want the detail for most or all of the items, then making them click to reveal the detail for each one only makes them do extra work
  • FAQs - FAQs have poor usability and usage
